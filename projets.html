Jeu en 2D

Type de projet : Projet scolaire Langages : C++ Outil(s) : CodeBlocks Travail en duo avec : Joel Jorge

Objectif : Concevoir un jeu 2D intégrant la gestion des personnages, des ennemis et des collisions.

Dans le cadre de ce projet, nous avons développé six classes en C++ afin d’organiser efficacement les différents composants du jeu : dictionnaire.cpp, image.cpp, niveau.cpp, objet.cpp, personnage.cpp, tuile.cpp.
Nous avons créé trois niveaux (deux de plus que ce qui était initialement demandé), en nous appuyant sur une grille structurée par des éléments solides ou non.
Le contrôle du personnage principal a été implémenté, ainsi que celui des ennemis : certains suivent un mouvement régulier, tandis qu’un ennemi adopte un comportement ciblé vers le joueur.
Un système de collecte de pièces a été ajouté : il est nécessaire de récupérer toutes les pièces pour ouvrir une porte et passer au niveau suivant.
Ce projet nous a permis d’approfondir notre compréhension du fonctionnement d’un jeu vidéo, notamment en ce qui concerne la gestion des entités, les collisions, et l’importance des mathématiques dans ce domaine.

Site web parc d'attraction

Type de projet : Scolaire Langages : HTML/CSS Outil(s) : Notepad++ Travail en duo : Mohamed BEN TALEB
Objectif de ce projet : Réaliser un site web (fictif) d'un lieu consacré au divertissement en duo
Nous avons donc avec mon binôme choisi de réaliser notre site web sur un parc d'attraction sur l'univers du célèbre manga/animé DRAGON BALL que nous apprécions particulièrement. Nous l'avons nommé Saiyan Land et avons arboré les couleurs de l'univers Dragon Ball sur tout le site avec du orange en couleur principale.
Ce site est composé de 4 pages : Accueil, Services, Tarifs et Map.
Toutes les pages sont composées d'une barre de navigation permettant de naviguer entre les différentes pages. On y retrouve également un bouton Réserver avec un formulaire de réservation (non fonctionnel).
Le footer, dont je suis le principal développeur, comporte une section réseaux sociaux et une autre avec des catégories qui ouvrent des pop-ups.
La page Accueil présente le parc avec texte, images, vidéos et informations de contact.
La page Activité (ma page) montre les attractions avec du texte, des images, des tableaux et une animation au survol.
La page Tarifs affiche des forfaits sous forme de tableaux stylisés.
La page Map permet de localiser le parc via un bouton "Visitez-nous".

Base de données d'une banque

Type de projet : Scolaire Langages : SQL Outil(s) : DB browser (SQLITE), Looping et Mocodo Travail en duo avec : Abdelmoumen HANNACHI
Objectif de ce projet : En partant d'un besoin exprimé par un client, étudier puis réaliser une base de données portant sur une partie du système d'information à développer.
Nous avons décidé de créer une base de données sur une banque et plus particulièrement sur le fonctionnement de la base de données des banques Crédit Agricole.
Pour cela, nous avons commencé par réaliser le Modèle Conceptuel de Données (MCD) puis le Schéma Relationnel (SR) .A partir de ces deux représentations, nous avons pu réaliser le dictionnaire des données.
Ensuite, nous avons débuter la création des tables en SQL : Ville, Agence, Type_CB, Type_Compte, Type_Transaction, Compte, Carte_Bancaire, Client et Transactions
Ensuite, nous avons inséré plusieurs données fictives dans la base de données pour chaque table (25 clients, 70 transactions...)
Pour finir, nous avons réaliser des requêtes (20 au total) afin d'intérroger la table et de renvoyer d'éventuelles données que des employés ou des clients de la banque auraient pu utiliser.
Note du projet : 15/20
